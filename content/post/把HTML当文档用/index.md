把 HTML 当文档用之后，PPT 和原型图都省了

先聊一下 HTML 这个东西的本质。

HTML 不是什么高深的东西。它就是一套带标签的纯文本。浏览器能读，AI 也能读。对 AI 来说，输出 HTML 跟输出一段文字几乎没有区别——语法简单、结构清晰、不需要额外工具。

但 HTML 有一个其他文本格式没有的优势：它直接是可视的。你在浏览器里打开一个 HTML 文件，它就是排版好的页面。什么软件都不用装。

这个特性让它在我现在的工具链里占了三个位置。

**当文档用**

我写东西用 md，但分享出去的时候经常转成 HTML。因为 HTML 不需要对方装任何软件——发个链接或者传个文件，浏览器打开就能看。排版固定、样式统一，不会像 docx 那样换个电脑格式全乱。

一些需要稍微精致一点的文档，直接在 md 里嵌一段 HTML 块。Obsidian 预览模式能正常渲染，效果跟网页一样。

**当 PPT 用**

装了一个 next-slide 的 skill（ClawHub 上有），效果是把 md 转成一套 HTML 幻灯片。写好的内容丢进去，几秒生成，50 多种主题可选。浏览器里打开就能演示，或者发链接给别人。

不用 Keynote，不用 PowerPoint，不用调格式。内容写在 md 里，换个壳就是幻灯片。

**当设计原型图用**

这是最让我觉得省事的一个用法。

以前做一个页面设计，要先在 Figma 里画一遍，然后写代码再实现一遍。现在直接写一段 HTML+CSS，跟 AI 说"帮我把这个页面布局做出来"，它输出一个能跑的 HTML 文件。打开看效果，不满意继续改。改到满意了，这段 HTML 可以直接当最终产品用——它不是原型图，它就是成品。

从原型到实现，一次完成。

**为什么会这样**

说穿了原因很简单：AI 最擅长输出的格式不是 docx、不是 PDF、不是 Figma 文件——是纯文本。而纯文本格式里，HTML 是表达能力最强的那个。它能排版、能交互、能做样式。你在 AI 面前写一个需求，它一秒给你一个能打开的页面。

不用打开 Figma、不用拖拽组件、不用调像素。

现在我的工作流变成了这样：想一个东西，用 md 记下来；需要给人看，转成 HTML；需要做演示，转成幻灯片；需要做设计，让 AI 直接生成 HTML 原型。同一份原料，换四个包装。

工具链就是 md + OpenClaw + next-slide。没什么门槛，试了就知道。
